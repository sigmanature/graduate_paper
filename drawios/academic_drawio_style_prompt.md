# 学术风格 draw.io 样式说明

适用对象：

- 第二章分层架构图
- 区间映射 / buffer\_head 对比图
- 第三章问题源头图
- 第四章总组件图

目标：

在 draw.io 中尽可能避免“业务系统图”“产品流程图”的观感，转为更接近中文学术论文插图的风格：规整、低饱和、强调层次和对应关系，而不是装饰性。

## 一、总体布局原则

1. 优先使用严格的横向分层或左右对照布局，不使用自由散点式摆放。
2. 同一层内的元素大小、间距和对齐方式必须一致。
3. 箭头优先垂直或水平正交连接，尽量避免弯曲和斜拉。
4. 一张图尽量只表达一个主问题，不把多个 unrelated 概念塞在一起。
5. 图题应简洁明确，图内说明不宜过长，长句放 caption，不放图中。

## 二、图形类型建议

1. 分层模块：圆角矩形
2. 数据对象（page、block、bvec、flash page）：普通矩形
3. 核心问题或冲突点：浅橙色圆角矩形，必要时可用六边形作为“结论节点”
4. 解决机制或改进点：浅绿色圆角矩形
5. 说明性标签：无边框文本或浅灰说明框

## 三、配色方案

建议保持白底，所有颜色低饱和，不使用紫黑渐变、强对比阴影或高亮荧光色。

### 主色

- 结构主色：`#4F81BD`
- 结构浅填充：`#DCEAF7`

### 次级结构色

- 次蓝色边框：`#5B9BD5`
- 次蓝色填充：`#EAF2FB`

### 问题/冲突色

- 边框：`#C55A11`
- 填充：`#FCE4D6`

### 解决/优化色

- 边框：`#70AD47`
- 填充：`#E2F0D9`

### 中性说明色

- 边框：`#7F7F7F`
- 填充：`#F3F3F3`

### 连线颜色

- 普通连线：`#7A869A`
- 强调连线：`#C55A11`

## 四、字体建议

1. 中文优先使用宋体、思源宋体、Noto Serif CJK SC 这类偏论文感字体。
2. 图标题可用黑体或思源黑体加粗。
3. 图中正文不要超过两级字号：
   - 标题：16~22
   - 层名 / 模块名：13~16
   - 注释：11~12
4. 不要使用花哨英文字体，不要混用过多字重。

## 五、线条与箭头

1. 边框线宽建议 1.2~1.8。
2. 连线线宽建议 1.0~1.3。
3. 箭头不要用 draw.io 默认大黑三角，优先细长 `blockThin` 风格或简洁实心箭头。
4. 如果是一组垂直映射关系，尽量用整齐的垂直箭头，不要让箭头交叉。
5. 若必须表示聚合，可用“分组标签 + 一根聚合箭头”，而不是 16 根线全部汇聚到一个点。

## 六、应避免的视觉问题

1. 避免厚重阴影、3D 效果、玻璃高光。
2. 避免渐变填充过重；大多数情况使用纯色浅填充即可。
3. 避免默认鲜艳绿色、鲜艳红色、鲜艳紫色混搭。
4. 避免图中出现太多代码函数名，除非该图本身就是调用流程图。
5. 避免元素边界不对齐，尤其是同层的小矩形。

## 七、适合本论文的具体风格关键词

如果用自然语言概括，可以把风格要求总结为：

“白底、低饱和蓝灰主色、浅橙标问题、浅绿标改进、严格对齐、正交箭头、层级清晰、少装饰、接近中文学术论文中的系统结构示意图，而不是互联网产品架构图。”
