# 函数名处理矩阵（工作稿）
## 一、函数名处理原则

| 类别 | 处理方式 | 例子 |
|---|---|---|
| 章节标题和核心算法锚点 | 保留函数名，但前面给中文职责 | 块映射到区间映射转换函数 `f2fs_set_iomap` |
| 通用框架入口 | 首次保留，后文用中文职责 | `iomap_readahead` → 通用预读入口 |
| F2FS 回调函数 | 第一次“中文职责（函数名）”，后文只用中文职责 | 缓冲读区间映射回调（`f2fs_buffered_read_iomap_begin`） |
| 内部辅助函数 | 尽量不进正文，必要时放括号或算法说明 | `map_is_mergeable` → 连续块合并判断 |
| 变量和字段 | 只在表格或必要机制处出现 | `m_lblk / m_pblk / m_len` 只在块映射状态表中解释 |
| 标志位/常量 | 表格化，不在正文反复堆叠 | `F2FS_MAP_MAPPED / NULL_ADDR / NEW_ADDR` |
| 代码细节宏 | 正文删除或下沉算法 | `LOCK_DONE` 不作为正文术语 |

## 二、函数名到中文描述替换矩阵

| 原函数/字段名 | 正文中文称呼 | 保留策略 |
|---|---|---|
| `f2fs_map_blocks` | F2FS 块映射查询入口 | 2.2、4.2 首次保留 |
| `f2fs_set_iomap` | 块映射到区间映射的统一转换函数 | 4.2.1 标题和首次保留 |
| `f2fs_buffered_read_iomap_begin` | 缓冲读区间映射回调 | 4.2.2 首次保留，后文用中文 |
| `f2fs_buffered_write_iomap_begin` | 缓冲写区间预分配回调 | 4.2.3、4.4.2 首次保留 |
| `f2fs_write_single_data_folio` | folio 子范围写回函数 | 4.5.1 首次保留，后文用中文 |
| `f2fs_write_cache_folios` | 脏 folio 遍历与分派入口 | 尽量少出现 |
| `f2fs_iomap_find_dirty_range` | 脏子范围查找逻辑 | 4.5.1 首次保留或直接中文 |
| `f2fs_compress_readpage_iter` | 压缩读内层迭代器 | 4.4.1 首次保留 |
| `f2fs_buffered_read_compress_iomap_begin` | 压缩读簇感知映射回调 | 4.4.1 首次保留 |
| `f2fs_iomap_finish_folio_read` | 统一读完成收敛函数 | 可在 4.4.1 完成计数段保留一次 |
| `f2fs_iomap_get_folio` | 预填充 folio 获取钩子 | 4.4.2 可改中文，不必多次写函数名 |
| `f2fs_ifs_alloc` | 扩展状态对象分配与迁移函数 | 4.3 首次保留 |
| `move_data_block` | GC 后台搬移中的数据块处理逻辑 | 4.6 首次保留或仅图/算法中出现 |
| `f2fs_submit_page_read` | GC 读路径子页 bio 提交逻辑 | 4.6 可改中文 |
| `f2fs_submit_page_bio` | GC 写路径子范围 bio 提交逻辑 | 4.6 可改中文 |
| `iomap_folio_state` | folio 内部块状态对象 | 第二章首次定义，第四章作为被扩展对象 |
| `f2fs_folio_state` | F2FS 扩展型 folio 内部块状态对象 | 第四章首次定义 |
| `read_bytes_pending` | 读完成待决字节计数 | 第二章定义，第四章使用 |
| `write_bytes_pending` | 写回完成待决字节计数 | 第二章定义，第四章使用 |
| `dirty_bytes_pending` | 压缩上下文脏字节待处理计数 | 第四章定义，专供压缩写回延迟解锁 |
