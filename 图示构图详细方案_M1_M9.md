# 图示构图详细方案（M1-M9）

说明：

1. 本文件用于把前面筛出的高优先级图示点，落实成可直接开画的构图方案。
2. 每张图都给出：用途、插入位置、推荐图种、版式、图形元素、箭头关系、建议标注、配色建议、避免事项。
3. 默认论文风格以“白底、低饱和蓝灰为主色、少量橙色强调问题、少量绿色强调改进”为原则。
4. 若在 Word 中排版，建议优先导出 `SVG`；如使用 TikZ，则最终转为高清 PDF/SVG 再插入。

统一配色建议：

- 主结构框：`#DCEAF7` 浅蓝填充，边框 `#4F81BD`
- 路径/机制框：`#E8F1FB` 更浅蓝填充，边框 `#5B9BD5`
- 问题/冲突框：`#FCE4D6` 浅橙填充，边框 `#C55A11`
- 改进/解决框：`#E2F0D9` 浅绿填充，边框 `#70AD47`
- 中性说明框：`#F3F3F3` 灰白填充，边框 `#7F7F7F`
- 主要箭头：深灰 `#595959`
- 强调箭头：橙色 `#C55A11`

统一图形建议：

- 分层/组件：圆角矩形
- 问题点/冲突点：六边形或高亮注释框
- 数据对象：普通矩形
- “例子中的数据范围”：长条矩形
- 计数器/状态位图：小矩形阵列
- 生命周期/流程步骤：顺序矩形 + 箭头

## M2 第二章系统分层架构图

### 这张图要回答什么

这张图专门回答：

`用户态 / VFS / page cache / F2FS / bio / FTL / flash` 七层分别是什么，它们如何围绕“同一段文件数据”传递抽象。

- `文件偏移`
- `页索引`
- `逻辑块号`
- `物理块号`

从上到下：

1. 用户态应用
2. 系统调用 / VFS
3. page cache / page descriptor
4. F2FS 映射层
5. bio / block layer
6. FTL
7. NAND flash

右侧单独拉一条“同一段 64KB 数据的表示方式”注释栏。

#### 第 1 层：用户态应用
框文字：
`应用 / read(fd, buf, 64KB)`

#### 第 2 层：系统调用 / VFS

#### 第 3 层：page cache

`页索引 index = offset / 4096`

#### 第 4 层：F2FS 映射层
框文字：
`F2FS / f2fs_map_blocks`
下方小字：
`文件偏移 → 逻辑块号 lbn → 物理块号 pbn`

可再放两个小中性框：

- 在 VFS：`文件字节区间 [0, 64KB)`

→ `LBA mapping`
→ `flash access`

## M3 一次 64KB buffered read 的完整路径流程图

### 插入位置

`第一二章重写稿.md` 第 139 行后，即 2.5 标题后。

### 这张图要回答什么

不是抽象分层，而是把一次具体访问走完：

`read(fd, buf, 64KB)` 到底如何一步步变成磁盘读取并返回。

### 推荐图种

典型 **顺序流程图 / 时序流程图**，`draw.io` 最合适。

### 布局

推荐 **横向泳道图**，从左到右五列：

1. 用户态
2. VFS / page cache
3. F2FS
4. block layer
5. device / FTL

时间从上到下。

### 泳道内容

#### 用户态泳道
顶部起点框：
`read(fd, buf, 65536)`

#### VFS / page cache 泳道
顺序框：
1. `generic_file_read_iter`
2. `按 offset 计算页索引`
3. `address_space 查找页缓存`
4. `cache miss`
5. `分配 page / folio`
6. `加锁 PG_locked`
7. `触发 readahead`
8. `收到完成回调后标记 PG_uptodate`
9. `拷贝回用户缓冲区`

#### F2FS 泳道
顺序框：
1. `f2fs_readahead / f2fs_map_blocks`
2. `offset → 逻辑块号`
3. `查节点树 / NAT`
4. `返回连续块映射`

#### block layer 泳道
顺序框：
1. `构建 bio`
2. `bio_add_page / 合并 bvec`
3. `submit_bio`
4. `bi_end_io 回调`

#### device / FTL 泳道
顺序框：
1. `LBA → 物理页映射`
2. `DMA 读取`
3. `返回完成`

### 关键标注

在 page cache 步骤边上放一个注释框：
`4KB 路径：分配 16 次 / 加锁 16 次`

在 block layer 步骤边上放一个注释框：
`bio 可合并为一次 64KB 提交`

最后在流程图底部放一个橙色结论框：
`块层已具备大块 I/O 提交能力，但上层仍按 4KB 对象多次组织`

### 不要画成什么

- 不要用纯组件框替代步骤
- 不要把这张图和 M2 合并
